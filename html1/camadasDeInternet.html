Camadas da Internet

O processo é muito parecido com o envio da carta

Aplication: O servidor está enviando , a aplicação seria tipo um web Server(servidor web) um exemplo o cliente estaria enviando um arquivo html. O mais alto nível

Transporte: é feito atravéss do protocolo TCP. O transporte é o 1° passo o endereçamento ou seja para onde vai.

Netowork(rede): é a camada do IP, ptotocolo de internet. é o transporte de um lugar para o outro.

Data Link: responsavel pela conexão de um com outro. A conexão do Data Link é a conexão de baixo nível, seria o ETHERNET, O cabo

Physical: é parte fisica, a transmissão dos dados.

DNS

Sistema de Nomes de Domínio (Domain Name System): O Sistema de Nomes de Domínio, ou servidor de nome de dominio mais conhecido pela nomenclatura em Inglês Domain Name System, é um sistema hierárquico e distribuído de gestão de nomes para computadores, serviços ou qualquer máquina conectada à Internet ou a uma rede privada.

Por exemplo eu quero acessar o www.example.com eu passo por todas as camadas da internet, e eu preciso do endereço do site e o endereço do site é identificado na camada IP, onde eu vou saber de qual ponto para qual ponto eu vou enviar, ou seja o IP, é o endereço do computador no meio de toda essa rede. Para poder acessar um site nós deveriamos saber o endereço IP, e isso é muito complicado por isso foi criado o DNS,para traduzir esse endereço de IP para um nome de dominio

TCP/IP

TCP/IP é a camada de Transporte

Lembrando que temos a camada de aplicação, e a camada de aplicação passa para camada de transporte.

Primeiro a camada de aplicação vai descobrir para onde irá mandar utilizando o DNS. Exemplo google.com tem o endereço de ip 8.8.8.8. Quem envia a informação é a camada de transporte , e essa informação não vai em linha reta. Pois como sabemos passamos por vários servidores para chegar ao endereço final. A conexão da internet é transportada em teia. O TCP quebra a informação em vários pacotes e transmite. O IP calcula chega no roteador, e elá ele calcula a rota mais rápida enviar esses pacotes. Aí o TCP para de enviar esses pacotes, quando recebe a informaçao de que o pacote foi recebido. E esse envio acontece diversas vezes. E se algum pacote foi perdido o TCP continua enviando o pacote, até ser sinalizado que foi recebido. O TCP do lado do cliente junta as informações e entrega para a aplicação. E é assim que é feita a transmissãode dados utilizando o TCP/IP.

Deep Web

Deep web e surface web conformam uma divisão do conteúdo da rede mundial de computadores quanto à indexação feita por mecanismos de busca padrão. Deep web corresponde à parte não indexada e surface web é a parte indexada.
