Progressive Web Apps – padrão ou palavra da moda?

Progressive Web Apps (PWA) é uma maneira moderna de construir aplicativos, incluindo os móveis, usando tecnologias puramente web e confiando apenas nos recursos e pontos fortes da plataforma web. Historicamente, foi a falta desses recursos na Web que causou o boom dos aplicativos móveis, usando as chamadas abordagens de desenvolvimento "nativas" (específicas da plataforma).

Site para verificar o suporte do PWA https://whatwebcando.today/

Surgido como um novo modelo de desenvolvimento de software, o Progressive Web App (PWA) transforma os navegadores (páginas web) em verdadeiras plataformas de apps!

Segundo a Wikipédia: Progressive Web App (PWA) é um termo usado para denotar aplicativos da web que usam as últimas tecnologias da web. Os aplicativos da web progressivos são páginas web (ou sites) tecnicamente regulares, mas podem aparecer ao usuário como aplicativos tradicionais ou aplicativos móveis (nativos). Este novo tipo de aplicativo tenta combinar os recursos oferecidos pela maioria dos navegadores modernos com os benefícios da experiência móvel.

Em resumo, é uma aplicação web com tecnologias que permitem termos a experiência de uso muito próxima da oferecida pelos mobile apps. As funcionalidades que estas tecnologias nos permitem são:

  • - Push Notification
  • - Ícone na tela home do smartphone
  • - Splash screen
  • - Processos rodando em background
  • - Suporte a funcionamento em modo Offline
  • - Acesso à camera e galeria
  • - Acesso à geolocalização
  • - Acesso à os contatos
  • - outros

Se o browswe não suportar um recurso, ele só não irá ter o recurso, mais o resto funciona. Instalar ou não é escolha do usuário. É usado pelo twiter, facebook, uber e outros...

Vantagens PWA(Progressive Web App)

  • - Poucas alterações fonte.
  • - Permite o uso de HTML/CSS/JavaScript
  • - Acesso á API'S como câmmera, microfone e localização
  • - Aplicações Leves
  • - Engajar o cliente em uma jornada. No primeiro acesso você resolve o problema dele e ponto. Os próximos acessos você monta uma jornada: pedindo as permissões de funcionalidades avançadas para o seu cliente, primeiro o push para ele receber notícias e afins, depois criar um ícone na home do smartphone, etc. Neste modelo o cliente não precisa ter a decisão de instalar ou remover o app. A medida que ele vai usando sua PWA ela vai se tornando cada vez mais parecida com um app.
  • - Envio de notificações push (alertas ou pop-up de sites ou apps)
  • - Pode ser usado offline
  • - Multiplataforma(IOS, Androide, SmartTvs, MacOS, Linux, Windowns)

Desvantagens do PWA:

  • - Não fica disponivel nas lojas do sistema operacional (android e IOS)
  • - Se a aplicação for pesada, pode ficar lento
  • - Acesso limitado ou inxexistente á alguns recursos:
  • - Pode funcionar de formas diferentes quando usa diferentes sistemas operacionais diferentes

App Shell

é o mínimo de HTML, CSS e javaScript que alimenta uma interface de usuário

Entre as premissas , estão:

  • - Promover Carregamento rápido
  • - Ter armazenamento em cache
  • - Ter exibição dinâmica de conteúdo

A arquiteura do app shell leva em consideração os seguintes pontos:

  • - Priorização do carregamento inicial, ou seja , o service worker , armazenará cache dos itens básicos para economizar tempo de carregamento;
  • - Carregamento assícrono visando já montar de forma rápida o aplicativo. Us de cache é bem vindo.
  • - Usar ferramentas confiáveis de gerenciamento de conteúdo estático para armazenar e atualizar cache

O shell do aplicativo é uma maneira de renderizar uma parte do seu aplicativo usando uma rota em tempo de compilação. Ele pode melhorar a experiência do usuário ao iniciar rapidamente uma página renderizada estática (um esqueleto comum a todas as páginas) enquanto o navegador baixa a versão completa do cliente e alterna para ela automaticamente após o carregamento do código.
Isso dá aos usuários uma primeira pintura significativa do seu aplicativo que aparece rapidamente porque o navegador pode renderizar o HTML e o CSS sem a necessidade de inicializar qualquer JavaScript.

Criar manifest json

O que é o arquivo manifest.json
O arquivo é um documento em JSON, nomeado como manifest. json, que fornece meta dados para a Progressive Web App. O arquivo fica armazenado no servidor onde a página está hospedada.
